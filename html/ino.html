3. Understanding the Arduino Sketch Structure üìì:

An Arduino program, commonly referred to as a sketch, consists of two main functions: setup() and loop(). Let's take a closer look at each of these functions:

setup() Function:

The setup() function is called once when the Arduino board is powered on or reset. It's where you initialize variables, configure pin modes, and set up any necessary peripherals. Here's an example:

void setup() {
    // Initialization code goes here
}

loop() Function:

The loop() function is where the main logic of your program resides. It's executed repeatedly as long as the Arduino board is powered on. Here's an example:

void loop() {
    // Main code goes here
}

By dividing your code into the setup() and loop() functions, you can organize your program's initialization and runtime logic effectively.

4. Working with GPIO Pins and Digital I/O ‚ö°Ô∏è:

GPIO (General Purpose Input/Output) pins allow you to interface with external devices and components. In Arduino C, you can control GPIO pins for digital input and output operations. Here's how you work with GPIO pins:

Initializing Pins:

Before using a pin, you need to specify its mode as either input or output. You can do this in the setup() function using the pinMode() function. For example: 

void setup() {
    pinMode(13, OUTPUT); // Set pin 13 as an output
}

Writing Digital Output:

To set the state of a digital output pin (e.g., to turn an LED on or off), you use the digitalWrite() function. For example: 

void loop() {
    digitalWrite(13, HIGH); // Turn on the LED connected to pin 13
    delay(1000); // Delay for 1 second
    digitalWrite(13, LOW); // Turn off the LED
    delay(1000); // Delay for 1 second
}

Reading Digital Input:

To read the state of a digital input pin (e.g., a push button), you use the digitalRead() function. For example:

void loop() {
    int buttonState = digitalRead(2); // Read the state of the button connected to pin 2
    if (buttonState == HIGH) {
        // Button is pressed
    } else {
        // Button is not pressed
    }
}

Understanding digital input and output operations is essential for interacting with external sensors, switches, and actuators.

5. Analog Input and Output üìä:

In addition to digital I/O, Arduino boards also support analog input and output operations. This capability allows you to interface with analog sensors and devices. Here's how you work with analog signals:

Reading Analog Input:

Analog sensors provide continuous voltage levels that can be measured by the Arduino's analog-to-digital converter (ADC). To read an analog input, you use the analogRead() function. For example:

void setup() {
    Serial.begin(9600); // Initialize serial communication
}

void loop() {
    int sensorValue = analogRead(A0); // Read analog input from pin A0
    Serial.println(sensorValue); // Print the sensor value to the serial monitor
    delay(1000); // Delay for 1 second
}

Generating Analog Output with PWM:

Pulse Width Modulation (PWM) allows you to simulate analog output using digital pins. PWM is commonly used for tasks like controlling LED brightness or motor speed. You can generate PWM signals using the analogWrite() function. For example: 

void setup() {
    pinMode(3, OUTPUT); // Set pin 3 as PWM output
}

void loop() {
    analogWrite(3, 128); // Set PWM duty cycle to 50% (0-255)
    delay(1000); // Delay for 1 second
}

By understanding analog input and output operations, you can interface with a wide range of sensors and actuators in your projects.

6. Interrupts üö®:

Interrupts in Arduino C allow you to respond to external events or signals without the need for continuous polling. When an interrupt occurs, the processor temporarily suspends its current execution to handle the interrupt, ensuring timely responses to critical events.

Configuring Interrupts:

Arduino boards support interrupts on specific GPIO pins. You can specify whether an interrupt should trigger on rising edges, falling edges, or both, depending on the nature of the event you want to detect.

void setup() {
    attachInterrupt(digitalPinToInterrupt(2), interruptHandler, CHANGE);
}

void loop() {
    // Main code here
}

void interruptHandler() {
    // Interrupt handler code here
}

In this example, an interrupt is configured on pin 2 to trigger on any change (both rising and falling edges). When the interrupt occurs, the interruptHandler function is called to handle the event.

Debouncing Mechanical Switches:

When using interrupts with mechanical switches or buttons, it's essential to debounce the input signal to prevent false triggering caused by mechanical bounce. Debouncing involves adding a delay or filtering mechanism to smooth out the signal and ensure reliable detection of button presses.

void interruptHandler() {
    delay(50); // Debounce delay
    // Interrupt handling code here
}

By incorporating debounce mechanisms into your interrupt handling logic, you can prevent false triggers and ensure accurate detection of external events.

7. Timers ‚è±Ô∏è:

Timers in Arduino C provide a versatile way to schedule tasks, generate PWM signals, or measure time intervals accurately. They are essential for various timing-related applications in embedded systems.

Configuring Timers:

Arduino boards support timers that can be configured to trigger periodic or one-shot events at specific intervals. You can specify the timer frequency, mode, and callback functions to execute tasks at predetermined times.

void setup() {
    Timer1.initialize(1000000); // Set timer interval to 1 second
    Timer1.attachInterrupt(timerCallback); // Attach interrupt handler
}

void loop() {
    // Main code here
}

void timerCallback() {
    // Timer callback code here
}

In this example, Timer1 is configured to trigger an interrupt every second. When the timer expires, the timerCallback function is called to execute the desired task.

By utilizing timers effectively, you can implement precise timing control and synchronization in your Arduino projects.

8. Serial Communication üì°:

Serial communication is a fundamental aspect of Arduino programming, allowing you to exchange data between the Arduino board and external devices such as computers, sensors, and displays.

Serial Communication Basics:

Arduino boards feature built-in serial communication ports (e.g., UART, USB) that enable bidirectional data transfer. You can use the Serial object to send and receive data over these ports.

void setup() {
    Serial.begin(9600); // Initialize serial communication at 9600 baud rate
}

void loop() {
    if (Serial.available() > 0) {
        char receivedChar = Serial.read(); // Read incoming byte
        Serial.print("Received: ");
        Serial.println(receivedChar); // Print received byte
    }
}

In this example, the Arduino board continuously reads incoming bytes from the serial port. When data is available, it reads the byte and prints it to the serial monitor.

Serial Communication Protocols:

Arduino supports various serial communication protocols, including UART, SPI, and I2C. These protocols enable communication with a wide range of devices and peripherals, such as sensors, displays, and microcontrollers.

#include <Wire.h>

void setup() {
    Wire.begin(); // Initialize I2C communication
}

void loop() {
    Wire.requestFrom(8, 6); // Request 6 bytes from slave device with address 8
    while (Wire.available()) {
        char receivedChar = Wire.read(); // Read incoming byte
        Serial.print(receivedChar); // Print received byte
    }
}

In this example, the Arduino board communicates with an I2C slave device by requesting data and reading the received bytes.

By mastering serial communication techniques, you can build complex and interactive Arduino projects that interface with external devices seamlessly.
